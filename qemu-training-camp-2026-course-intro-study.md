## 各阶段在讲什么

### 1. 导学阶段

🎥 讲者：导学阶段包含 3 类内容：
- 编程基础：`C 语言训练营`、`Rust 入门课程`
- 体系结构：`GPU 发展历史`、`GPGPU 学习资料`、`RISC-V 体系结构`
- QEMU 入门：开发环境搭建、构建文档、RISC-V 模拟器开发入门、大模型知识库辅助学习

🎥 讲者：导学阶段资料可以作为后续阶段的长期补充资源，不设晋级门槛，自学完成即可进入基础阶段。

🧠 我：这一段你不要按“新手补课”学法来。
- 如果你 `C` 已经比较稳，就不要在 C 语法上耗时。
- `Rust` 可以先当预习，不要一上来把注意力拉偏。
- 真正值得你吃透的是：`QEMU 环境搭建`、`构建文档`、`RISC-V 模拟器入门`。

🧠 我：导学阶段对你最有价值的定位不是“学习新内容”，而是“补齐训练营自己的语言体系和资料入口”。它相当于后面所有阶段的索引页。

✅ 你现在就做：
- 把训练营网页结构、资料入口、博客入口都存成自己的资料地图。
- 优先完成 `QEMU 开发环境搭建` 和 `构建/运行最小示例`。
- 遇到不会的地方，不要硬扛，直接把问题丢给我，我帮你转成可执行步骤。

### 2. 基础阶段

🎥 讲者：基础阶段的课程内容包括：
- 虚拟机的发展历史与基本工作原理
- QEMU 常用启动参数与命令行
- QEMU 启动流程与初始化机制
- `QOM (QEMU Object Model)` 对象模型
- `MemoryRegion` 内存区域管理
- QEMU 常用调试手段

🎥 讲者：配套实验是 `C` / `Rust` 二选一，完成其一即可进入专业阶段。

🧠 我：这是你最该认真吃的第一块核心内容。
- `启动参数` 让你知道命令行和代码如何连起来。
- `启动流程` 让你知道 QEMU 是怎么“活起来”的。
- `QOM` 让你知道设备对象是怎么组织的。
- `MemoryRegion` 让你知道地址空间和 MMIO 是怎么落到模型里的。
- `调试手段` 决定你后面做实验时会不会迷路。

🧠 我：很多人会觉得这部分“基础，不够酷”，但后面你真做外设、板级或驱动联动项目时，最值钱的恰恰是这一段。因为它回答的是：
- 一个设备怎么被创建
- 一个寄存器访问怎么进入模型
- 出问题时去哪里下断点

✅ 你现在就做：
- 把这一阶段当成“QEMU 心智模型建立期”。
- 实验建议优先选 `C` 方向，除非你已经确定后面要走 `Rust for QEMU`。
- 学这一段时，每一章都要追到源码入口，不要只看讲义。

### 3. 专业阶段

🎥 讲者：专业阶段分成 4 个方向：
- `CPU 建模`
  - RISC-V 体系结构与 CPU 建模
  - `TCG` 二进制动态翻译引擎
  - 指令翻译流程 `Guest -> IR -> Host`
  - `SoftMMU`
  - 多核调度与 `MTTCG`
- `外设建模`
  - `Machine / Board` 建模流程
  - `Clock`
  - `Interrupt`
  - `PCIe`
  - 硬件外设建模实战
- `GPGPU 建模`
  - GPGPU 体系结构概述
  - `SIMT` 执行模型
  - `BAR / MMIO / DMA / MSI-X`
  - Kernel 调度与执行后端
  - `QTest`
- `Rust for QEMU`
  - Rust FFI 与 QEMU 绑定
  - Rust 外设建模实践
  - Rust 测试框架

🎥 讲者：专业阶段考核是 `四选一` 完成一个实验方向；此外还需要写一篇博客总结学习过程，才能晋级项目阶段。

🧠 我：如果从“对你以后做项目最有帮助”来排优先级，我建议：
1. `外设建模`
2. `CPU/TCG` 的核心理解
3. `GPGPU 建模`
4. `Rust for QEMU`

🧠 我：原因很简单。
- 你已有内核/驱动背景，最容易和你现有能力接起来的是 `外设建模`。
- `CPU/TCG` 很重要，但更像“理解执行引擎”，不一定是你第一批动手最多的地方。
- `GPGPU` 很有潜力，但它更偏特定方向。
- `Rust for QEMU` 很前沿，但先把 QEMU 主线打通更重要。

🧠 我：这里我尤其建议你不要只把“选一个实验方向”当作过关条件，而要把它当作“项目阶段前的试投产”。换句话说，你选的专业阶段方向，最好能和你后面的项目方向连续。

✅ 你现在就做：
- 专业阶段优先主攻 `外设建模`。
- 同时补看 `CPU/TCG`，至少建立边界感，知道执行链路和设备链路在哪里交汇。
- 博客不要拖到最后写，边学边记，最后只是整理，而不是从零回忆。

### 4. 项目阶段

🎥 讲者：项目阶段目前展示了 4 个项目方向：
- `CXLMemSim + QEMU GPU 可观测系统`
- `勘智 K230 卫星星务计算单元建模`
- `Wine-CE RISC-V 适配 x86 应用`
- `大模型 Agent 自动化外设建模`

🎥 讲者：项目阶段的晋级/结营要求是提交项目报告和代码仓库，由项目导师考核。

🧠 我：如果只考虑“你以后做系统/虚拟化/驱动方向的含金量”，我会这样看：
- `K230`：最像正统系统建模项目，和板级、SoC、外设、内核协同都贴得近。
- `Agent 自动化外设建模`：最前沿，适合你后面把 AI 辅助建模也纳入自己的方法论。
- `CXLMemSim + QEMU GPU`：技术含量高，但跨度更大，对前置知识要求也高。
- `Wine-CE RISC-V 适配 x86`：更偏二进制翻译/兼容层思路，路线和前三个不完全一样。

🧠 我：站在你的背景上，我最推荐两个优先选择：
- 第一优先：`K230`
- 第二优先：`Agent 自动化外设建模`

✅ 你现在就做：
- 项目阶段别追“最酷”，要追“最能和你现有能力形成复利”。
- 如果你想以后更偏硬件软件协同，就优先 `K230`。
- 如果你想把 AI + 建模能力作为个人特色，就优先 `Agent 自动化外设建模`。

## 晋级规则怎么理解

🎥 讲者：晋级体系大致是：
- 导学 -> 基础：不设门槛，自学完成即可
- 基础 -> 专业：完成 `C` 或 `Rust` 编程实验其一
- 专业 -> 项目：完成一个方向实验 + 一篇博客总结
- 项目 -> 结营：提交项目报告 + 代码仓库

🧠 我：这套规则其实很合理，它在逼你完成 3 种最重要的输出：
- `动手实验`
- `知识表达`
- `项目交付`

🧠 我：很多人只重视第一项，最后做过但说不清；也有人只会写总结，做不出东西。训练营这套设计是希望你两边都补上。
